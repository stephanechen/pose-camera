## 一、产品概述

### 1.1 产品定位

一款**AI智能取景参照拍照工具**，帮助用户精准复刻小红书等平台的网红照片。

```
┌─────────────────────────────────────────────────────────────────┐
│                    产品核心价值主张                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   📸 一键上传参考图 → 小红书/相册图片直接导入                      │
│                                                                 │
│   ✨ AI智能透明化 → 参考图变透明，不遮挡真实取景                    │
│                                                                 │
│   🎯 精准构图复刻 → 叠加参照，对齐人物/场景位置                    │
│                                                                 │
│   📱 简单易上手 → 无需复杂设置，看一眼就会用                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 用户场景

| 场景 | 描述 | 用户痛点 |
|-----|------|---------|
| 咖啡馆打卡 | 看到小红书博主在某个咖啡馆拍的美照 | 不知道怎么摆姿势、怎么取景 |
| 旅行拍照 | 想要复刻某个网红旅行打卡照 | 不知道人物放在哪里、背景怎么选 |
| 闺蜜合照 | 想拍出博主同款闺蜜照 | 构图总是差一点，效果差很多 |
| 探店拍照 | 想拍出网红餐厅同款美食照 | 不知道俯拍角度、光线怎么找 |

### 2.2 功能模块详解

#### 2.2.1 图片导入模块

**功能描述**：支持多种方式导入参考图

| 导入方式 | 操作流程 | 限制 |
|---------|---------|------|
| 相册选择 | 调起微信相册选择图片 | 支持JPG/PNG/WEBP，单张≤20MB |
| 拍照导入 | 调用相机拍摄参考图 | 用于现场拍摄参考图 |
| 剪贴板 | 识别剪切板中的图片 | 从小红书截图复制 |

**技术要点**：
- 微信小程序wx.chooseMedia API
- 图片压缩预处理（保持比例，最大边1920px）
- 读取EXIF信息（方向、尺寸）

#### 2.2.2 AI透明化模块

**功能描述**：将参考图进行透明化处理，生成可叠加的半透明图层

**处理方式**：

| 方案 | 描述 | 优点 | 缺点 |
|-----|------|-----|------|
| **纯前端Alpha透明** | 直接调整PNG透明度 | 无需联网，效果即时 | 无法智能识别主体 |
| **AI语义分割（推荐）** | AI识别主体与背景 | 主体保留，背景透明 | 需要调用AI接口 |

**推荐方案 - AI语义分割**：
```
原始图片 ──→ 主体检测 ──→ 背景分离 ──→ Alpha融合 ──→ 透明图层
                │                        │
                ▼                        ▼
           人物/物品识别              边缘羽化处理
```

**透明度调节**：
- 范围：10% - 90%
- 默认值：50%
- 步进：5%
- 支持滑块拖动 + 快捷按钮

#### 2.2.3 取景叠加模块

**功能描述**：将透明参考图与相机实时画面叠加显示

**核心功能**：

| 功能 | 说明 | 操作方式 |
|-----|------|---------|
| 参考图显示/隐藏 | 切换参考图显示状态 | 点击眼睛图标 |
| 参考图缩放 | 调整参考图大小匹配取景框 | 双指捏合 |
| 参考图拖拽 | 移动参考图位置 | 单指拖动 |
| 参考图旋转 | 微调参考图角度 | 两指旋转 |
| 重置位置 | 恢复参考图初始状态 | 双击重置 |

**叠加渲染**：
```
┌─────────────────────────────────────────┐
│           相机预览层 (底层)               │
│  ┌─────────────────────────────────┐    │
│  │                                 │    │
│  │     实时相机画面                 │    │
│  │                                 │    │
│  │                                 │    │
│  └─────────────────────────────────┘    │
│                                         │
│           参考图层 (顶层)                │
│  ┌─────────────────────────────────┐    │
│  │  ░░░ 透明参考图 ░░░             │    │
│  │   ░░░░░░░░░░░░░░░░             │    │
│  │    ░░░░░░░░░░░░░░░             │    │
│  └─────────────────────────────────┘    │
│                                         │
└─────────────────────────────────────────┘
```

#### 2.2.4 辅助对齐模块

**功能描述**：提供辅助线帮助用户更精准地对齐

**辅助工具**：

| 工具 | 描述 | 适用场景 |
|-----|------|---------|
| 三分线 | 显示三分法构图线 | 人物/主体位置对齐 |
| 对角线 | 显示对角构图参考 | 引导线构图 |
| 中心点 | 显示画面中心十字 | 中心构图 |
| 网格线 | 显示均匀网格 | 整体比例参考 |
| 人形轮廓（高级） | AI识别参考图中人物，生成轮廓线 | 人物姿势模仿 |

**人形轮廓功能**：
```
参考图 ──→ 人形检测 ──→ 骨骼关键点 ──→ 轮廓生成
              │                          │
              ▼                          ▼
         头/肩/手等                 简化轮廓线条
         关键位置识别               (半透明白色)
```

#### 2.2.5 拍照保存模块

**功能描述**：拍摄最终照片并保存

**拍照选项**：

| 选项 | 说明 |
|-----|------|
| 参考图出镜 | 最终照片是否包含透明参考图 |
| 滤镜预设 | 应用预设滤镜提升效果 |
| 原图保存 | 同时保存无滤镜原图 |

**保存流程**：
```
拍照 ──→ 显示预览 ──→ 选择保存选项 ──→ 保存至相册
         │                              │
         ▼                              ▼
      重新拍摄                       保存成功提示
```

## 四、交互设计

### 4.1 手势操作

| 手势 | 操作对象 | 操作效果 |
|-----|---------|---------|
| 单指点击 | 参考图 | 显示/隐藏操作菜单 |
| 单指拖动 | 参考图 | 移动参考图位置 |
| 双指捏合 | 参考图 | 缩放参考图大小 |
| 双指旋转 | 参考图 | 旋转参考图角度 |
| 双击 | 参考图 | 重置参考图位置和大小 |
| 长按 | 参考图 | 调出更多选项（删除/替换） |

### 4.2 快捷操作

| 操作 | 触发方式 | 效果 |
|-----|---------|------|
| 隐藏参考图 | 点击眼睛图标 | 临时隐藏，方便取景 |
| 切换参考图 | 点击相册图标 | 重新选择参考图 |
| 快速调透明度 | 滑块滑动 | 实时预览透明度变化 |
| 快速拍照 | 点击快门 | 立即拍摄 |

### 4.3 状态反馈

| 场景 | 反馈形式 |
|-----|---------|
| 参考图加载中 | 加载动画 + 提示"正在处理..." |
| AI处理中 | 进度条 + "AI正在分析图片..." |

### 5.2 核心实现方案

#### 5.2.1 相机预览 + 透明叠加

**方案**：使用Canvas进行多层渲染

```javascript
// 渲染层级结构
Layer 1 (底层): <camera> 相机原生组件
Layer 2 (中层): <canvas> 相机画面截图（用于融合）
Layer 3 (顶层): <canvas> 透明参考图层（可交互）
Layer 4 (覆盖): <view> UI控制组件
```

**实现要点**：
- 使用`wx.createCameraContext`获取相机帧
- Canvas 2D绑定到camera组件区域
- 参考图使用`drawImage`绘制，支持透明度
- 实时渲染需要保持30fps以上

#### 5.2.2 AI透明化处理

**方案A - 纯前端Alpha通道（轻量级）**：
```javascript
// 直接调整PNG透明度
function applyTransparency(imageData, alpha) {
  const data = imageData.data;
  for (let i = 3; i < data.length; i += 4) {
    data[i] = Math.round(data[i] * alpha);
  }
  return imageData;
}
```

**方案B - 云端AI语义分割（推荐）**：
```javascript
// 调用云端API进行主体分割
async function processImage(imagePath) {
  const res = await wx.cloud.callContainer({
    path: '/api/segment',
    imagePath: imagePath
  });
  return res.data.transparentImage;
}
```

#### 5.2.3 拍照融合

```javascript
async function takePhoto() {
  // 1. 获取相机原图
  const cameraCtx = wx.createCameraContext();
  const cameraPhoto = await cameraCtx.takePhoto();
  
  // 2. 获取Canvas上的叠加结果
  const overlayData = canvas.toDataURL('image/png');
  
  // 3. 融合两张图片（根据用户选择）
  const finalPhoto = await mergeImages(
    cameraPhoto.tempImagePath, 
    overlayData,
    { includeReference: userChoice }
  );
  
  // 4. 保存
  wx.saveImageToPhotosAlbum({ filePath: finalPhoto });
}
```

## 九、风险与合规

### 9.1 技术风险

| 风险项 | 风险描述 | 应对方案 |
|-------|---------|---------|
| Canvas性能 | 多图层渲染可能导致卡顿 | 优化渲染逻辑，使用离屏Canvas |

|-------|---------|---------|
